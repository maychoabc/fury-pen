# 一致性审查规则

> S5 一致性审查的标准执行流程和校验方法。
> S5 每章完成后自动执行，按 genre-config.md 的 check_priority 逐层校验。

---

## 执行流程

### 步骤 1：加载类型配置

1. 读取项目 `bible/genre-config.md`
2. 提取 `check_priority` 列表（含每层的 name、level、sources、description）
3. 如 genre-config.md 不存在，使用默认通用校验（仅人物性格+伏笔状态两层，均为 SOFT）

### 步骤 2：逐层校验

对 check_priority 中的每一层，按顺序执行校验：

**每层校验的通用步骤**：

1. 读取该层 `sources` 列出的所有 bible 文件
2. 读取刚完成的新章节文本（`chapters/ch{N}.md`）
3. 比对：新章节内容是否与 source 文件中的规则/事实矛盾
4. 记录比对结果

### 步骤 3：比对方法（按层类型）

---

#### 第1层：规则不违背（规则怪谈 HARD）

**比对逻辑**：
- 从 `core-rules.md` 和 `instances/{当前副本}.md` 提取所有规则条文
- 扫描新章节中每个角色行为/事件
- 判断：是否有任何行为/事件直接违反了某条规则？

**判定标准**：
- 角色明确违反规则条文 → **违规**
- 角色找到规则的漏洞/第三种解法 → **不违规**（但需记录创意解法供后续参考）
- 规则本身是误导规则，角色违反了误导规则而遵守了真实规则 → **不违规**

**特殊情况处理**：
- 规则冲突优先级：如 core-rules.md 定义了冲突优先级，按优先级判定
- 模糊规则：如规则表述有歧义，记录为"待确认"而非直接判违规

---

#### 第2层：信息一致性（HARD）

**比对逻辑**：
- 从 `characters/{角色名}.md` 的「已知信息集」提取每个出场角色的当前已知信息
- 扫描新章节中每个角色的推理/判断/决策
- 判断：角色是否基于他还没获取的信息做出了判断？

**判定标准**：
- 角色使用了已知信息集中不存在的信息做推理 → **违规**
- 角色凭直觉/猜测做出了碰巧正确的判断（但逻辑链不依赖未知信息） → **不违规**
- 角色被告知了新信息并在本章使用 → **不违规**（但 S4 应已提取此信息到已知信息集）

**检查顺序**：
1. 读取出场角色的已知信息集
2. 标记每条推理/判断依赖的信息来源
3. 交叉比对：依赖的信息是否在已知信息集中

---

#### 第3层：数值连续性（SOFT）

**比对逻辑**：
- 从 `progression.md` 取当前积分/技能等级/道具库存
- 扫描新章节中涉及数值的描写
- 判断：数值是否与前章末状态连续？

**判定标准**：
- 积分/道具数量在本章开头与上章末不一致 → **记录差异**（SOFT放行）
- 技能等级凭空提升无对应事件 → **记录差异**
- 有明确消耗/获取事件但与 progression.md 记录不匹配 → **记录差异**

---

#### 第4层：轮回记忆一致性（规则怪谈 SOFT）

**比对逻辑**：
- 从 `timeline.md` 取轮回结构记录
- 扫描新章节中角色的记忆引用
- 判断：角色的记忆是否符合轮回设定（保留/清除）

**判定标准**：
- 设定"轮回保留记忆"但角色遗忘了上一轮事件 → **记录差异**
- 设定"轮回清除记忆"但角色引用了上一轮信息 → **违规**（但 SOFT 放行）
- 设定"部分保留"（如保留情绪但不保留细节）→ 比对具体保留规则

---

#### 第5层：伏笔状态（SOFT）

**比对逻辑**：
- 从 `foreshadowing.md` 取所有伏笔条目
- 检查：预计回收章≤当前章的"待回收"伏笔是否已回收
- 检查：新埋设的伏笔是否与已有伏笔冲突/重复

**判定标准**：
- 应回收但未回收的伏笔 → **提醒**（SOFT放行，但提示作者注意）
- 新伏笔与已有伏笔功能重复 → **提醒**
- 误导伏笔被角色提前识破（不符合角色已知信息）→ **升级为第2层问题**

---

#### 通用层：人物性格漂移（所有类型 SOFT）

**比对逻辑**：
- 从 `characters/{角色名}.md` 取性格三词
- 扫描新章节中角色的言行
- 判断：角色的行为是否偏离性格三词锚定？

**判定标准**：
- 行为与三词完全矛盾且无合理解释（如"隐忍"角色突然暴怒） → **记录差异**
- 行为偏离三词但有弧光推进解释 → **不违规**
- 偏离程度超过"性格锚定说明"中的偏离容忍范围 → **记录差异**

---

### 步骤 4：记录结果

将所有校验结果写入 `tracking/consistency-log.md`：

```
## 第 {N} 章一致性审查

审查时间：{当前时间}

| 优先级 | 校验层 | 结果 | 矛盾描述 | 建议修复 |
|--------|-------|------|---------|---------|
| HARD | 规则不违背 | ✅通过 / ❌违规 | {具体描述} | {建议} |
| HARD | 信息一致性 | ✅通过 / ❌违规 | {具体描述} | {建议} |
| SOFT | 数值连续性 | ✅通过 / ⚠️差异 | {具体描述} | {建议} |
| SOFT | 轮回记忆 | ✅通过 / ⚠️差异 | {具体描述} | {建议} |
| SOFT | 伏笔状态 | ✅通过 / ⚠️提醒 | {具体描述} | {建议} |
| SOFT | 人物性格 | ✅通过 / ⚠️差异 | {具体描述} | {建议} |
```

### 步骤 5：硬违规拦截

- 如任何 HARD 层校验结果为 ❌违规：
  1. **中止流程，不进入下一章**
  2. 标记当前章节需要重写
  3. 写入 `tracking/rewrite-history.md` 记录违规详情
  4. 自动触发 S7 修订重写流程（如 S7 已实现）
  5. 或提示用户："第{N}章存在硬违规：{违规描述}。建议重写本章或调整设定。"

### 步骤 6：软违规放行

- 所有 SOFT 层的差异/提醒：
  1. 记录到 consistency-log.md
  2. **继续流程，进入下一章**
  3. 在记录中标注"待后续统一修订"

---

## 审查报告输出格式

完成所有层校验后，向用户输出审查摘要：

```
📋 第 {N} 章一致性审查结果

硬校验（一票否决）：
  ✅/❌ 规则不违背 — {结果摘要}
  ✅/❌ 信息一致性 — {结果摘要}

软校验（记录放行）：
  ✅/⚠️ 数值连续性 — {结果摘要}
  ✅/⚠️ 轮回记忆 — {结果摘要}
  ✅/⚠️ 伏笔状态 — {结果摘要}
  ✅/⚠️ 人物性格 — {结果摘要}

总判定：✅ 通过，可继续下一章 / ❌ 存在硬违规，需重写
```
