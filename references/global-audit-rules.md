# S8 全局终审规则（Global Final Audit）

> 当整本小说完成（或用户主动说"全局审查"/"全书终审"/"final audit"）时触发。
> 与 S5（逐章审查）不同，S8 是**跨章节、跨 bible 的全局扫描**，专门捕获只有"看完全书"才能发现的系统性问题。
> 取 AI_NovelGenerator 的"终稿一致性扫描"之长，并弥补 S5 逐章视角的盲区。

---

## 触发条件

- 用户说"全局审查"/"全书终审"/"一致性总检"/"final audit"
- 或 `progress.md` 显示"已完成章节 = 总预计章节"且用户确认收尾
- 或用户指定审查范围："审查第X-Y章"

---

## 执行流程

### 步骤 1：加载全集

1. 读取 `bible/` 全部模块（meta / genre-config / outline / characters/* / worldbuilding/* / instances/* / foreshadowing / progression / lorebook / style-guide）
2. 读取 `chapters/` 全部正式版章节（按文件名排序）
3. 读取 `tracking/consistency-log.md`、`tracking/rewrite-history.md`、`tracking/global-audit-log.md`（历史）

### 步骤 2：全局维度扫描

按以下 10 个全局维度逐一扫描（维度可随类型在 genre-config 中裁剪）：

#### 维度 A：孤儿伏笔（最常被忽略）
- 扫描 `foreshadowing.md` 中所有状态=待回收 / 部分回收 的条目
- 检查其"预计回收章"是否 ≤ 全书最后一章
- 若预计回收章已过全书末尾仍未回收 → **孤儿伏笔**（硬编码问题）
- 输出：{编号} {内容} 应在第{X}章回收但全书未见 → 建议补写回收 or 调整状态为"已推翻(误报)"

#### 维度 B：数值时间线完整性
- 以 `progression.md` 变动记录为基准，重建积分/物资/等级的全书曲线
- 检查：是否存在"断点"（某章末值 ≠ 下一章开头值且无事件解释）
- 检查：最终章的库存/等级是否与正文描写一致

#### 维度 C：信息时间线完整性（角色知情权）
- 对每个主要角色，按其「已知信息集 → 后续获取」的时间戳，重建"第K章时他知道什么"
- 回扫全书：是否存在某章中该角色使用了"第>K章才获取"的信息（跨章信息泄漏）
- 这是 S5 单章视角的盲区，S8 才能发现

#### 维度 D：角色弧光连续性
- 读取每个角色的"动机与弧光"与"状态变更记录"
- 检查：弧光走向（成长/堕落/坚守/觉醒）是否在全书事件中得到支撑
- 检查：是否有"中途人设突变"无对应转折事件

#### 维度 E：规则跨章自洽
- 汇总 `core-rules.md` + 所有 `instances/*.md` 规则条文
- 扫描全书：是否有后文"悄悄改了"前文确立的规则（规则漂移）
- 特别检查：规则冲突优先级在全书是否一贯适用

#### 维度 F：时间线/行程合理性
- 以 `timeline.md` 为轴，检查多线叙事的时间戳是否冲突
- 检查：角色能否在同一时间出现在两个地点（除非设定允许）

#### 维度 G：伏笔密度与节奏
- 统计全书伏笔埋设/回收比
- 标记：连续 N 章无任何伏笔推进（节奏塌陷）or 单章伏笔过载
- 输出节奏建议（非错误，供作者优化）

#### 维度 I：角色出场连续性（v3 新增）
- 以 `characters/_index.md` + 各角色出场章节记录为基准，统计每个核心角色的"最长连续缺席章数"
- 阈值警戒：
  - 主角/主要视角角色连续缺席 > 3 章 → 标记"角色掉线"，建议补充桥段或旁白交代
  - 配角/重要辅助角色连续缺席 > 5 章 → 标记中优先级提醒
  - 排除"已死亡/已退场"角色
- 输出：{角色名} 从第{X}章到第{Y}章连续缺席{N}章 → 建议补充剧情或确认是否合理

#### 维度 J：命名一致性（v3 新增）
- 从 bible 中提取全部专有名词，建标准命名对照表：
  - 地名（worldbuilding/locations.md）
  - 人名（characters/*.md）
  - 科技术语（worldbuilding/rules.md、core-rules.md）
  - 舰船/武器/组织名
- 回扫全书章节正文，用正则/模糊匹配查找非标准变体：
  - 例："沉默之巢"标准 → 正文中出现"沉默巢"、"沉默之城"、"Silent Nest中文译名不同"等变体
  - 例："心波武器"标准 → 正文中出现"心波武器系统"、"Mind-Scour"混用
  - 例：人名全称/简称不一致——"韩寂"vs"韩教授"（合理变体）、"严理鸣"vs"导师"（合理变体）
- 将**合理变体**加入标准命名表（接受别名），将**非合理变体**标记为需统一修正

### 步骤 3：生成全局审查报告

写入 `tracking/global-audit-log.md`，并向用户输出摘要：

```markdown
📑 全书一致性终审报告（共 {M} 章）

🔴 必须修复（阻断发布）：
  - [孤儿伏笔] F012 应在第40章回收，全书未见 → 第{X}章补写
  - [信息泄漏] 主角第12章使用了第25章才获得的信息
  - [规则漂移] 第30章违背了第5章确立的元规则R-03

🟡 建议优化（不影响发布）：
  - [节奏] 第18-22章伏笔推进空白
  - [弧光] 配角乙动机支撑不足

🟢 通过项：
  - 数值时间线连续 / 角色弧光闭环 / 规则跨章自洽

总判定：❌ 存在 {n} 项阻断问题，修复后可发布 / ✅ 可发布
```

### 步骤 4：修复引导

- 对每项阻断问题，给出**具体修复建议 + 命中章节**
- 提供两种修复路径：
  - A. 局部补写/修订 → 触发 S7（针对具体章节）+ S4+S5 重跑
  - B. 调整设定 → 修改 bible 后重跑受影响章节的 S2→S5
- 修复后建议再跑一次 S8 确认清零

---

## 与 S5 的边界

| 维度 | S5（逐章） | S8（全局） |
|------|-----------|-----------|
| 视角 | 单章 vs bible | 全书 vs bible |
| 擅长 | 本章规则/信息/数值 | 孤儿伏笔/跨章信息泄漏/规则漂移 |
| 触发 | 每章自动 | 收尾或用户主动 |
| 输出 | 逐章 consistency-log | global-audit-log |
