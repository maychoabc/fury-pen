## 一致性校验优先级

S5 一致性审查时按此优先级逐层校验：

1. 规则不违背
   level: HARD（一票否决，发现即打回重写）
   sources: [core-rules.md, instances/当前副本.md]
   description: 新章节中任何角色行为/事件，不得违反 core-rules.md 的元规则或当前副本的特有规则。如"副本中禁止大声说话"则绝不能出现角色大声说话的情节。

2. 信息一致性
   level: HARD（一票否决）
   sources: [characters/*/已知信息]
   description: 角色做出的判断、推理、决策，只能基于他"已知信息"集合中的内容。不能用还没获得的信息破局。如主角在第3副本才摸清的规则，第2副本中绝不能提前使用。

3. 数值连续性
   level: SOFT（记录但放行）
   sources: [progression.md]
   description: 积分/技能等级/道具库存前后对得上。如上一章末积分1000，本章开头不能变成800（除非有明确消耗事件）。

4. 轮回记忆一致性
   level: SOFT（记录但放行）
   sources: [timeline.md]
   description: 如果本轮重启了，上一轮角色"学到"的规则记忆是否按设定保留或清除。如设定"轮回保留记忆"则不能出现角色遗忘上一轮事件。

5. 伏笔状态
   level: SOFT（记录但放行）
   sources: [foreshadowing.md]
   description: 检查伏笔是否按计划回收。规则怪谈的伏笔往往是"规则暗示"——某条看似无用的规则在后续副本中才是通关钥匙。

## 初始化 Q&A 定制追加

以下问题叠加在通用问题之上，按阶段分组：

### Phase 1 额外问题（核心机制）

叠加在通用"书名/一句话核心/视角/基调/篇幅"之后追问：

1. "轮回/副本结构是什么？时间循环式（十日轮回）？空间推进式（无限副本）？还是两者结合？"
2. "主角如何进入副本？自愿/被迫/选拔/意外？能否退出？"
3. "死亡条件：在副本里怎样会死？死亡=真死，还是重启本轮（保留/清除记忆）？"
4. "规则范式：规则是'明示'（系统公布条文让角色遵守）还是'暗示'（角色靠摸索/试错推断）？请给1-2条规则雏形。"
5. "★关键：规则冲突优先级——当两条规则表面矛盾时，以什么为准？（如'禁止开门'vs'必须离开房间'→后者优先/前者是误导/需要找到第三种解法）这个优先级直接决定一致性闸门的判定逻辑。"
6. "成长系统：有没有积分/技能/道具？跨副本是否保留？数值膨胀节奏如何？"

### Phase 2 额外问题（人物）

叠加在通用"主角性格三词/配角"之后追问：

1. "主角的信息获取方式：靠观察？靠系统提示？靠其他角色告知？靠试错？"
2. "是否有'信息差'设计——某些角色知道主角不知道的信息？"

### Phase 3 额外问题（首个副本）

叠加在通用"首章冲突/出场人物/悬念"之后追问：

1. "第一个副本的核心规则集（3-5条）——请逐条列出"
2. "通关条件是什么？"
3. "副本中要埋设的规则暗示伏笔——哪些规则看似无用但后续副本中才是钥匙？"

### Phase 4 额外问题（长线悬念）

叠加在通用"世界真相/总篇幅/结局走向"之后追问：

1. "系统的真相/起源——谁设计了这些副本？为什么？（读者慢揭开，但作者现在确定）"
2. "预计多少个副本？每个副本大约多少章？"
3. "最终通关的条件——是解开所有规则？找到系统起源？还是其他？"

## 判定锚点（正反例）★v2 新增：用具体样例锚定各层判定，提升闸门可复现性

> S5 按 check_priority 校验时，对照以下正/反例判定"是否违规"，减少模糊空间。

### 第1层 规则不违背（HARD）
- ✅ 正例：副本规则"夜间不可点灯"，角色全程摸黑、靠听觉脱险 → 合规
- ❌ 反例：同一副本中角色"点亮手机照明找路" → 直接违反，HARD 拦截打回
- ⚠️ 边界：角色发现"灯油是诱饵（误导规则），真实规则是不能出声"，于是点灯但不出声 → 不违规，记录创意解法

### 第2层 信息一致性（HARD）
- ✅ 正例：主角第5副本才从系统提示获知"规则R-07真实含义"，此前章节从未用该含义做判断
- ❌ 反例：第3副本主角"凭R-07真实含义破解机关"，但该信息第5副本才获得 → 信息越界，HARD 拦截
- ⚠️ 边界：角色凭直觉猜对（逻辑链不依赖未知信息）→ 不违规

### 第3层 数值连续性（SOFT）
- ✅ 正例：上章末积分1200，本章开头1200，中段消耗300→900，与 progression 一致
- ❌ 反例：上章末1200，本章开头突变为800且无消耗事件 → 记录差异

### 第4层 轮回记忆一致性（SOFT）
- ✅ 正例：设定"轮回清除记忆"，新轮回主角不记得上轮细节
- ❌ 反例：设定清除记忆，主角却引用"上轮某NPC说的话" → 记录差异

### 第5层 伏笔状态（SOFT）
- ✅ 正例：F003 预计第8章回收，第8章兑现
- ⚠️ 提醒：F003 预计第8章回收，第9章仍未回收 → 提醒作者注意回收窗口
