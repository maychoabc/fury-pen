## 一致性校验优先级

1. 信息层级一致性
   level: HARD（一票否决）
   sources: [characters/*/已知信息, worldbuilding/lore.md]
   description: 悬疑推理对"谁知道什么"极其敏感。角色（包括侦探）的推理只能基于已知线索。真相信息只能由作者掌控，角色不能意外获得超出线索链的真相片段。

2. 时间线逻辑
   level: HARD（一票否决）
   sources: [worldbuilding/timeline.md]
   description: 案发时间线必须严密无矛盾。死者的行踪、证人的证词、物证的时间戳，全部前后一致。

3. 伏笔/线索回收
   level: SOFT（但高度重视）
   sources: [foreshadowing.md]
   description: 悬疑推理的伏笔分两类：线索伏笔（指向真相，必须回收）和误导伏笔（指向错误方向，必须被推翻）。两类都要追踪状态。

4. 角色动机一致性
   level: SOFT
   sources: [characters/*/动机与弧光]
   description: 每个角色的行为必须有动机支撑。嫌疑人的犯罪动机、侦探的追查动机，不能无故改变。

5. 逻辑推理链完整性
   level: SOFT
   sources: [foreshadowing.md]
   description: 侦探的推理过程是否基于已展示的线索，而非"突然灵感"。

## 初始化 Q&A 定制追加

### Phase 1 额外

1. "案件类型：谋杀？失踪？盗窃？连环案？密室？还是多种叠加？"
2. "叙事策略：读者跟侦探同步推理？还是读者先看到真相再看侦探追查（倒叙推理）？"
3. "真相复杂度：单层真相？双层真相（表面真相背后还有真相）？还是三层？"
4. "误导设计：有多少条误导线索？指向什么错误方向？"

### Phase 2 额外

1. "侦探的推理风格：逻辑型（一步一步推导）？直觉型（灵感闪现）？混合？"
2. "嫌疑人数量和动机预设：列出所有嫌疑人的初始动机假设"

### Phase 3 额外

1. "案件开场的呈现方式：发现尸体？接到委托？目击事件？"
2. "第一章要展示的核心线索是什么？"
3. "第一章要埋设的误导线索是什么？"

### Phase 4 额外

1. "真相的全貌——谁做的？为什么？怎么做？（作者确定，读者慢揭）"
2. "推理链的关键转折点——侦探从哪个线索开始推翻表面真相？"
3. "结局：真相大白？留有悬念？开放式？"

## 判定锚点（正反例）★v2 新增

### 第1层 信息层级一致性（HARD）
- ✅ 正例：侦探第8章才从物证获知"凶手是左撇子"，此前推理从未用过该信息
- ❌ 反例：第3章侦探就"精准描述凶手惯用手"但当时无任何线索支持 → 信息越界，HARD 拦截
- ⚠️ 边界：侦探凭"现场血迹方向"合理推测（线索已展示）→ 不违规

### 第2层 时间线逻辑（HARD）
- ✅ 正例：死者死亡时间22:00，所有嫌疑人该时段行踪前后一致
- ❌ 反例：证人证词"22:30见到死者"，但尸检死亡时间20:00 → 矛盾，HARD 拦截

### 第3层 伏笔/线索回收（SOFT，高重视）
- ✅ 正例：第2章展示的"打火机"在第15章成为定罪物证
- ❌ 反例：全文埋了线索却从未兑现，或误导伏笔未被推翻 → 记录差异

### 第4层 角色动机一致性（SOFT）
- ✅ 正例：嫌疑人动机（债务）贯穿始终
- ❌ 反例：嫌疑犯突然换动机无铺垫 → 记录差异

### 第5层 逻辑推理链完整性（SOFT）
- ✅ 正例：结论每一步都有已展示线索支撑
- ❌ 反例：侦探"突然灵光一现"得出结论，无线索链 → 记录差异
