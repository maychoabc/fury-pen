# S3 写前自检规则（Self-Check）

> S3 生成草稿后、写入 `ch{N}-draft.md` 之前，由 Agent 内部执行的一次"预审查"。
> 目的：在草稿交给用户审阅 / 进入 S4→S5 之前，提前拦截 HARD 级违规，减少无谓的重写轮次。
> 这是 S5 闸门的前移版本——S5 是"事后审查+记录"，本规则是"写后即时自检+就地修复"。

---

## 何时执行

在 S3 步骤 2（生成章节正文）产出草稿后、步骤 3（写入草稿文件）之前，执行一次自检：

1. 取 S2 已组装的上下文提示词（含所有约束、已知信息、规则、数值、伏笔、文风）
2. 逐条对照下方清单扫描刚生成的草稿
3. 输出一份**自检报告**（内部，不强制展示给用户，除非发现需用户决策的问题）
4. 若发现 HARD 级问题 → **就地重写问题段落**（保留无问题的精彩段落），重新自检，直到通过或最多 2 次
5. 自检通过后，再写入 `ch{N}-draft.md` 并进入用户审阅

> 自检最多自修复 2 轮；仍不通过则照常写入草稿并交给 S5 处理（不阻塞流程）。

---

## 自检清单（按优先级）

### ★ HARD-1：规则不违背（规则怪谈/末世/异能等类型）

- [ ] 草稿中每个角色行为/事件，是否违反 `core-rules.md` 或当前副本规则条文？
- [ ] 是否出现"角色用了尚未在本章/前文合法获得的破局手段"？
- [ ] 模糊规则：有无表述歧义处被角色利用了"第三种解法"？→ 允许，但标记创意解法留痕

**判定**：任何明确违反 → HARD 违规，必须修复对应段落。

### ★ HARD-2：信息一致性（所有类型核心）

- [ ] 每个出场角色的推理/判断/决策，是否只依赖其「已知信息集」中的内容？
- [ ] 有无角色"突然知道"了 S4 尚未登记、且本章未告知的信息？
- [ ] 跨副本/跨章信息：第 K 章才摸清的规则，是否在第 <K 章提前使用？

**判定**：角色基于未知信息破局 → HARD 违规。

### HARD-3（按类型）：数值/物资边界

- 规则怪谈/都市异能：`progression.md` 当前积分/技能等级/道具，本章开头是否突变（无消耗/获取事件）？
- 末世生存：物资库存/基地等级/人口，是否无来源地增减？
- 悬疑推理：时间线是否出现前后矛盾的时间戳？

**判定**：无事件支撑的数值跳变 → HARD 违规。

### SOFT 自检（记录但放行，与 S5 同层，此处提前标注以便 S5 不重复）

- [ ] 角色言行是否偏离性格三词（无弧光支撑）？
- [ ] 预计回收章≤N 的伏笔是否在本章回收/推进？
- [ ] 禁用词清单中是否有词出现？→ 必须消除（禁用词零容忍，视为硬性）
- [ ] 文风：视角/叙事距离/节奏是否偏离 style-guide？

> 注：禁用词出现虽在原设计归为"文风约束"，但属作者硬性要求，自检中发现应**直接清除**，不进入用户审阅稿。

## 自检报告格式（内部）

```markdown
## 第{N}章 写前自检

自检轮次：第1轮
| 级别 | 检查项 | 结果 | 问题段落 | 修复动作 |
|------|-------|------|---------|---------|
| HARD | 规则不违背 | ✅/❌ | {段落} | {已就地修复/待S5} |
| HARD | 信息一致性 | ✅/❌ | {段落} | ... |
| HARD | 数值边界 | ✅/❌ | ... | ... |
| SOFT | 性格漂移 | ✅/⚠️ | ... | 记录 |
| SOFT | 伏笔推进 | ✅/⚠️ | ... | 记录 |
| 硬性 | 禁用词 | ✅/❌(已清除) | {词} | 已替换 |

结论：✅ 通过，写入草稿 / ⚠️ 第X轮修复后通过 / ❌ 转 S5 处理
```

---

## 与 S5 的分工

- **本规则（写前自检）**：Agent 内部即时修复，目标是"交给用户的草稿已不犯 HARD 错"
- **S5（一致性审查）**：正式、留痕、跨章比对，处理 SOFT 差异并做最终放行/拦截判定
- 自检中已修复并标注的 SOFT 项，S5 不再重复报；自检未覆盖的全书级问题，由 S5 和 S8 兜底
